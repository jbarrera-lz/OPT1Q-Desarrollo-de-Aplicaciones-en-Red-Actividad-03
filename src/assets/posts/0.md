## ¿Qué es Angular?
---

Se trata de un framework para construir aplicaciones web en ```HTML```, ```CSS``` y ```JavaScript``` y/ó ```TypeScript```.

## ¿Por qué utilizar Angular?
---

En grandes aplicaciones web, desarrollos hechos principalmente en ```JavaScript``` o ```JQuery``` son bastantes difíciles de mantener en el largo plazo. Además, el propio testeo de los mismos no es tarea fácil, lo cuál implica que la integración de una nueva funcionalidad sea un tarea bastante compleja. Es por ello que en los últimos años han surgido una serie de _frameworks_ para desarrollo web, entre los que cabe destacar:

+ Angular
+ React
+ Vue.js

En el caso de ```Angular```, se pueden destacar las siguientes ventajas en su uso:

- Proporciona a nuestra solución una estructura ordenada.
- Fácil de entender y de mantener.
- Ecosistema de librerías bastante amplio y disponible de manera gratuita.
- Fácil de testear de manera automatizada.

## Breve definición de la arquitectura de una aplicación desarollada con Angular
---

Angular se centra principalmente en lo que se denomina el ```User Interface``` o ```Front End``` de nuestra aplicación, es decir, en lo que se representa de manera visual en el navegador web. En lo que se denomina el ```Back End``` es donde residen los elementos que entre otras tareas, pueden realizar el almacenamiento de los datos, o a grosso modo, es donde se ejecuta la lógica de negocio. 

Pero para poder realizar la comunicación entre ```Front-End``` y ```Back-End``` se utiliza lo que se denomina servicios HTTP o APIs (Application Programming Interface), que permita establecer mecanismos de comunicación entre el cliente web y el ```endpoint```.
